## Usage

```bash
unideploy init [flags]
```

## What it does

`unideploy init` runs UniDeploy's 15-rule security checker on the files in your current working directory and streams findings to the dashboard in real time.

1. The CLI creates a session and prints a 6-digit code.
2. Open [unideploy.in/connect](https://unideploy.in/connect) and enter the code.
3. Findings stream to both your terminal and the dashboard simultaneously.

## Flags

| Flag | Description |
|------|-------------|
| `--json` | Output results as structured JSON, skip the interactive session. |
| `--ci` | Exit with code `1` on any CRITICAL findings. Skips the interactive session. |
| `--local` | Point at `localhost:8000` instead of the production API. |

## JSON / CI mode

When `--json` or `--ci` is passed, the session is skipped entirely — scan runs offline and exits immediately.

```bash
unideploy init --json
```

```json
{
  "project_meta": {
    "framework": "Next.js",
    "file_count": 84,
    "scan_duration_ms": 312
```

## Apply auto-fixes

After any scan, run:

```bash
unideploy fix
```

UniDeploy rewrites the flagged lines in-place. Use `--dry-run` to preview changes without writing files:

```bash
unideploy fix --dry-run
```

## When to use init vs scan

| | `unideploy init` | `unideploy scan` |
|-|-----------------|-----------------|
| Source | Local files | GitHub URL |
| Output | Terminal + dashboard | Terminal |
| Private repos | Yes | Requires repo access |
| CI/CD | `--ci` flag | `--ci` flag |
| Speed | Instant (local) | ~30s (remote clone) |
