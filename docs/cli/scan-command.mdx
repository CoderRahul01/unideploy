---
title: "scan"
description: "Scan a GitHub repository by URL (remote scan mode)."
---

## Usage

```bash
unideploy scan [github_url] [flags]
```

If `github_url` is omitted, UniDeploy reads the remote URL from your local git config (`git remote get-url origin`).

<Note>
For scanning **local files** with a live dashboard session, use [`unideploy init`](/cli/init-command) instead. The `scan` command triggers a remote GitHub scan via the UniDeploy backend.
</Note>

## Flags

| Flag | Description |
|------|-------------|
| `--branch <branch>` | Branch to scan. Defaults to `main`. |
| `--ci` | Exit with code `1` if any CRITICAL findings are found. |
| `--json` | Output results as structured JSON. |
| `--local` | Point at `localhost:8000` instead of the production API. |

## Exit codes

| Code | Meaning |
|------|---------|
| `0` | Scan completed, no CRITICAL findings. |
| `1` | CRITICAL findings present (only when `--ci` is used). |
| `2` | Scan failed — network or API error. |

unideploy scan https://github.com/your/repo --json
