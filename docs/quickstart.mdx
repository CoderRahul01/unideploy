## Install

```bash
npm install -g unideploy
```

Requires Node.js 18 or higher. Verify the installation:

```bash
unideploy --version
# unideploy/0.2.3
```

## Run a scan

Navigate to your project root and run:

```bash
cd your-project
unideploy init
```

UniDeploy runs 15 security rules across your local files and opens a live session on the dashboard:

```

  Framework: Next.js detected
  Scanning 84 files...
```

## Apply auto-fixes

```bash
unideploy fix
```

UniDeploy rewrites the flagged lines in-place — CORS wildcards, missing security headers, console.log leaks. Review the diff before committing.

## Reading the output

| Field | Meaning |
|-------|---------|
| `[CRITICAL]` | Immediate data exposure risk — hardcoded secret, .env committed to git |
| `[HIGH]` | Exploitable without authentication — missing RLS, no auth on API routes |
| `[MEDIUM]` | Best-practice gaps — missing headers, unvalidated input, debug mode |
| Grade | A–F. Any CRITICAL finding → grade F. |
| Risk Score | 0–100. `CRITICAL × 40 + HIGH × 15 + MEDIUM × 5 + LOW × 1`, capped at 100. |

## CI / offline mode

Skip the interactive session and get structured JSON output:

```bash
unideploy init --json
```

```json
{
  "project_meta": { "framework": "Next.js", "file_count": 84, "scan_duration_ms": 312 },
  "findings": [...],
  "summary": {
    "total": 14,
    "critical": 1,
    "high": 8,
    "medium": 5,
    "low": 0,
    "grade": "F",
    "risk_score": 100
  }
}
```

Use `--ci` to exit with code `1` on any CRITICAL finding — gate your merges:

```bash
unideploy init --ci
```
